# 1. 표지

## 제목
- **Text-Conditioned Lightweight Navigation World Model**
- **텍스트 융합을 통한 NWM 경량화 및 OOD 일반화 연구**

## 부제
- 저해상도 visual world modeling에서 텍스트 기반 semantic prior의 효과 검증

## 표지 하단
- 발표자 이름
- 소속 / 연구실
- 날짜

---

# 2. 문제 배경

## 2.1 Navigation World Model의 가능성
- NWM은 과거 관측과 행동을 바탕으로 미래 시각 관측을 생성하는 모델
- 단순 정책 네트워크와 달리, trajectory simulation 기반 planning이 가능
- 제약 조건이 있는 navigation에도 유연하게 대응 가능

## 2.2 하지만 실제 적용에는 한계가 있음
- 모델이 크고 추론 비용이 큼
- 고해상도 입력과 긴 autoregressive rollout이 필요
- OOD 환경에서는 context를 잃고 hallucination/mode collapse가 발생할 수 있음

## 2.3 핵심 질문
- **텍스트 정보를 semantic prior로 주입하면**
  - 저해상도에서도 성능을 유지할 수 있는가?
  - 새로운 환경에서 일반화를 개선할 수 있는가?
  - 전체 시스템 효율을 해치지 않고 실시간성에 기여할 수 있는가?

---

# 3. 연구 동기

## 3.1 왜 텍스트를 쓰는가
- 이미지 해상도를 낮추면 세부 시각 정보가 빠르게 손실됨
- 반면 텍스트는 장면의 의미를 압축적으로 전달 가능
- 예: “앞에 열린 복도”, “오른쪽에 장애물”, “좌측 전방으로 진입 가능”

## 3.2 기대 효과
- 저해상도 입력이 잃어버린 semantic cue 보완
- OOD 환경에서 scene understanding 보조
- planning 시 더 안정적인 trajectory ranking 가능성

## 3.3 현실 제약
- 텍스트를 위해 더 무거운 모델을 붙이면 실용성이 사라짐
- 따라서 **“배보다 배꼽이 커지지 않는”** 경량 텍스트 파이프라인이 필수

---

# 4. 연구 목표

## 4.1 최종 목표
- NWM에 텍스트 조건을 결합한 **Text-Conditioned Lightweight NWM** 설계

## 4.2 세부 목표
- 텍스트가 world model의 보조 prior로 작동하는지 검증
- 저해상도 경량화에서 텍스트가 성능 저하를 상쇄하는지 측정
- OOD 환경에서 텍스트가 일반화에 기여하는지 분석
- end-to-end latency 기준으로 실제 시스템 효율성 평가

## 4.3 연구 가설
- 고해상도 in-domain에서는 텍스트 이득이 제한적일 수 있음
- 그러나 **저해상도**와 **OOD 환경**에서는 텍스트가 의미 보존에 유의미한 도움을 줄 가능성이 큼

---

# 5. 핵심 아이디어

## 5.1 기본 아이디어
- 시각 입력만으로 부족한 semantic context를 텍스트로 보완
- 텍스트를 NWM의 conditioning stream에 함께 주입
- 텍스트는 주행 정책 자체가 아니라 **scene prior / goal prior** 역할 수행

## 5.2 설계 원칙
- 기존 pretrained NWM을 최대한 보존
- 구조 변경은 최소화
- 텍스트 조건은 필요할 때만 작동하도록 점진적으로 학습
- 최종 시스템은 반드시 경량 추론 경로를 가질 것

## 5.3 실험 전략
- **Phase 1:** 원본 해상도에서 개념 증명
- **Phase 2:** 단계적 해상도 축소를 통한 본격 경량화
- **Phase 3:** 실시간 추론 파이프라인 검증

---

# 6. 전체 시스템 개요

## 6.1 오프라인 학습 파이프라인
- 비디오 데이터셋에서 프레임 추출
- 무거운 VLM으로 고품질 보조 캡션 생성
- CLIP text encoder로 텍스트 임베딩 추출
- 이미지, action, time, text embedding을 함께 사용해 NWM 파인튜닝

## 6.2 온라인 추론 파이프라인
- 현재 프레임 입력
- 초경량 모델로 텍스트 태그/짧은 설명 생성
- 텍스트 임베딩을 NWM에 입력
- trajectory prediction / planning / ranking 수행

## 6.3 핵심 포인트
- 무거운 VLM은 **학습용 데이터 생성에만 사용**
- 실제 추론에서는 **초경량 semantic extractor**만 사용

---

# 7. 사용 모델

## 7.1 Base World Model
- 논문 기반 NWM의 smallest variant 사용
- 초기 baseline과 PoC 실험의 기준 모델

## 7.2 Text Encoder
- CLIP Text Encoder 사용
- 텍스트를 시각-의미 정렬이 잘 된 embedding space로 변환

## 7.3 Offline Caption Generator
- Qwen2-VL 7B 또는 LLaVA 계열 사용
- 학습 데이터에 대해 고품질 보조 캡션을 대량 생성

## 7.4 Online Lightweight Semantic Extractor
- 1차 후보: **YOLOv10 Nano**
- 2차 후보: **Moondream2**
- 우선순위는 YOLO 기반 태그 방식
- richer text가 꼭 필요할 때만 경량 VLM 확장 검토

# 8. 사용 데이터셋

## 8.1 In-domain 학습 및 평가
- **RECON**
  - outdoor 중심
  - NWM 논문에서 핵심 평가 데이터셋
- 필요 시 **SCAND** 추가
  - indoor/outdoor 혼합
  - 동적 장면과 사회적 내비게이션 요소 포함

## 8.2 OOD 평가
- **GO Stanford**
- 새로운 환경에서 world model generalization 측정
- 텍스트가 context 유지와 hallucination 완화에 도움이 되는지 평가

## 8.3 데이터 선택 이유
- 원 논문과 유사한 조건에서 비교 가능
- in-domain / out-of-domain 성능 차이를 분리해 해석 가능

# 9. 오프라인 캡션 생성 파이프라인

## 9.1 프레임 추출
- 비디오에서 1 FPS 기준으로 이미지 추출
- 필요 시 key frame sampling 추가 검토

## 9.2 캡션 생성
- heavy VLM으로 장면 의미를 설명하는 보조 캡션 생성
- 예: 통로, 문, 장애물, 방향성, 장면 구조

## 9.3 텍스트 정보 설계
- **Scene-only text**
  - 장면 구조, 장애물, 통로 정보
- **Goal-related text**
  - 목표 방향 또는 목표와의 상대 위치
- **Scene + Goal text**
  - 두 정보를 결합한 형태

## 9.4 저장 방식
- 텍스트를 CLIP embedding으로 변환 후 사전 저장
- 학습 시 텍스트 인코더를 매 step 돌리지 않음
- 학습 병목 최소화

---

# 10. 모델 아키텍처 수정

## 10.1 기본 방향
- 기존 NWM의 conditioning 구조에 텍스트 embedding 추가
- action, temporal shift, diffusion timestep과 함께 사용

## 10.2 1차 결합 방식
- text projection 후 conditioning vector에 결합
- pretrained NWM 구조를 크게 바꾸지 않는 방향

## 10.3 권장 결합 방식
- 단순 합산보다 **gated fusion** 우선 검토
- 텍스트 영향력을 학습적으로 조절
- 초기에는 거의 base NWM처럼 동작하도록 설계

## 10.4 안정성 장치
- text projection layer zero initialization
- 초기 학습 안정화
- 기존 NWM 파괴 방지

# 11. 학습 전략

## 11.1 Phase 1: 원본 해상도 PoC
- 원본 또는 기준 해상도 유지
- 텍스트가 실제로 도움이 되는지 먼저 검증
- 구조 변경 효과와 학습 안정성 확인

## 11.2 Phase 2: 단계적 해상도 축소
- 해상도를 한 번에 크게 줄이지 않음
- 예: high → 128 → 112 → 64
- 각 단계에서 텍스트가 성능 저하를 얼마나 보완하는지 평가

## 11.3 학습 안정화 기법
- text branch zero-init
- text dropout 적용
- 일부 샘플에서 텍스트를 비워 robust하게 학습

## 11.4 왜 단계적으로 가는가
- 실패 원인을 분리하기 쉬움
- “저해상도 때문인지”, “fusion 문제인지”, “text 품질 문제인지” 해석 가능

# 18. 예상 결과

## 18.1 기대 시나리오
- 고해상도 in-domain에서는 텍스트 이득이 크지 않을 수 있음
- 저해상도에서는 텍스트가 성능 하락을 유의미하게 완화할 가능성
- OOD에서는 scene semantics 보조 효과가 더 크게 나타날 가능성

## 18.2 이상적인 결론
- low-res text-conditioned NWM이
  - low-res no-text NWM보다 확실히 좋고
  - 일정 조건에서 high-res base NWM에 근접하며
  - 더 나은 효율성을 보임

## 18.3 최종 메시지
- 텍스트는 world model의 주된 입력이 아니라
- **경량화와 일반화를 돕는 보조 semantic prior**로 기능함

# 19. 예상 리스크와 대응 방안

## 19.1 리스크 1: 텍스트가 별 도움이 없을 수 있음
- 고해상도 in-domain에서는 시각 정보만으로 충분할 수 있음
- 대응: OOD와 low-resolution setting에 초점을 맞춰 해석

## 19.2 리스크 2: 캡션이 shortcut이 될 수 있음
- goal 정보를 과도하게 주면 unfair advantage 가능
- 대응: observation-only text와 goal-related text를 분리 평가

## 19.3 리스크 3: 경량 텍스트 추출기 비용이 더 클 수 있음
- online semantic extractor가 전체 효율을 해칠 수 있음
- 대응: YOLO 기반 태그 추출을 기본안으로 설정하고 E2E latency로 검증

## 19.4 리스크 4: noisy text에 취약할 수 있음
- 실제 환경에서 tag miss / false positive 발생 가능
- 대응: text dropout, noisy tag augmentation으로 robust training

- NWM에 텍스트 조건을 추가해
- 저해상도와 OOD 상황에서 semantic prior를 제공하고
- 경량화와 일반화를 동시에 달성할 수 있는지 검증

## 21.2 핵심 기여 예상
- text-conditioned navigation world model 설계
- resolution-efficiency-generalization trade-off 분석
- 실시간 시스템 관점의 end-to-end 평가

## 21.3 마무리 메시지
- 본 연구는 “멀티모달 조건 추가” 자체보다
- **실시간 embodied world model을 위한 의미 기반 경량화 전략**을 제안하는 데 의의가 있음
